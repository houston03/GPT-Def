Контекстный менеджер

Объект логику которого определяет момент входа в блок и выхода из него, используя методы (синхронные __enter__, __exit__)(ассинхронные __aenter__, __aexit__)

Самым популярным КМ является класс с функцией Open внутри, ведь она гарантирует закрытие файла, даже если в блоке есть ошибка, тем самым сохранив ресурс ПК.

Пример:

Открытие и закрытие файлов:

with open('file.txt', 'r') as file:
content = file.read()
print(content)


Создание собственных контекстных менеджеров:

class MyResource:
def __enter__(self):
print("Открытие ресурса")
# Инициализация ресурса
return self
# Возвращаем себя, чтобы использовать в блоке `with`
def __exit__(self, exc_type, exc_val, exc_tb):
print("Закрытие ресурса")
# Очистка ресурса
return False
# Возвращаем `False`, чтобы передать исключение внешнему коду

Итераторы и генераторы

Генератор

Генераторы - это функция, которая с помощью выражения yield генерирует последовательность значений, также реализует такие методы как __iter__() и __next__(). Так как в памяти генератора находится один элемент, итерировать может только его(сохранив больше ОП).

Итератор

Итератор реализует такие методы как __iter__() и __next__(), iter-возвращает итератор, а next-возвращает следущий элемент последовательности, после вызывает исключение StopIteration

Пример:

Генератор:

def squares(n):
for i in range(n):
yield i ** 2
for square in squares(5):
print(square)


Итератор:

class MyIterator:
def __init__(self, data):
self.data = data
self.index = 0
def __iter__(self):
return self
def __next__(self):
if self.index < len(self.data):
value = self.data[self.index]
self.index += 1
return value
else:
raise StopIteration
my_list = [1, 2, 3, 4, 5]
my_iterator = MyIterator(my_list)
for item in my_iterator:
print(item)

Методы __new__, __init__

__new__:

__new__ обрабатывает создание объекта. Вызывается при вызове имени класса

__init__:

__init__ обрабатывает инициализацию созданного объекта. Вызывается после возвращения экземпляра класса в __new__, передавая его в __init__ в качестве параметра self.

Пример:

class MyCustomClass:
def __new__(cls, *args, **kwargs):
print("Вызывается __new__")
instance = super().__new__(cls)
return instance
def __init__(self, name, age):
print("Вызывается __init__")
self.name = name
self.age = age
person = MyCustomClass("Иван", 25)
print(person.name) # Вывод: Иван
print(person.age) # Вывод: 25

Декораторы

Декоратор - это функция, которая принимает функцию с таким же именем, но с более расширенным функционалом, часто используется во фреймвроках.

Пример:

def log_function_call(func):
def wrapper(*args, **kwargs):
print(f"Вызывается функция {func.__name__} с аргументами: {args}, {kwargs}")
result = func(*args, **kwargs)
print(f"Функция {func.__name__} возвращает результат: {result}")
return result
return wrapper
@log_function_call
def add(x, y):
return x + y
print(add(2, 3))
log_function_call(func): Эта функция принимает другую функцию в качестве аргумента и возвращает новую функцию wrapper.

Объектно-ориентированное программирование

Абстрация:

Скрытие внутренних характеристик функции от пользователя.

Полиморфизм:

Использование одной функции для разных форм(типов данных).

Наследование:

Способ создания класса. Функциональность нового класса(дочернего) наследнуется от уже существующего класса(базового)

Инкапсуляция:

Ограничение доступа к переменным, методам, для предотвращения модификации данных.

Пример:

Наследование:

class Animal:
def __init__(self, name):
self.name = name
def speak(self):
print(f"{self.name} говорит!")
class Dog(Animal):
def speak(self):
print(f"{self.name} лает!")
class Cat(Animal):
def speak(self):
print(f"{self.name} мяукает!")
my_dog = Dog("Рекс")
my_cat = Cat("Мурка")
my_dog.speak() # Вывод: Рекс лает!
my_cat.speak() # Вывод: Мурка мяукает!


Абстракция:

from abc import ABC, abstractmethod
class Shape(ABC):
@abstractmethod
def area(self):
pass
class Circle(Shape):
def __init__(self, radius):
self.radius = radius
def area(self):
return 3.14 * self.radius ** 2
class Rectangle(Shape):
def __init__(self, width, height):
self.width = width
self.height = height
def area(self):
return self.width * self.height
circle = Circle(5)
rectangle = Rectangle(4, 6)
print(circle.area()) # Вывод: 78.5
print(rectangle.area()) # Вывод: 24


Инкапсуляция:

class BankAccount:
def __init__(self, balance):
self.__balance = balance
def deposit(self, amount):
self.__balance += amount
def withdraw(self, amount):
if self.__balance >= amount:
self.__balance -= amount
print("Снятие успешно")
else:
print("Недостаточно средств")
def get_balance(self):
return self.__balance
account = BankAccount(1000)
account.deposit(500)
account.withdraw(200)
print(account.get_balance()) # Вывод: 1300

Базы данных и индексы

Виды БД:

1)Реляционные базы данных-Данные хранятся в таблицах с определенной структурой (строки и столбцы), где столбцы имеют определенный тип данных.Используют SQL (Structured Query Language) для запросов, обновления и управления данными.Обеспечивают целостность данных с помощью ограничений (PRIMARY KEY, FOREIGN KEY, UNIQUE), что гарантирует консистентность и точность информации.
2)Нереляционные базы данных (NoSQL)-Не требуют жесткой структуры таблиц. Данные могут храниться в разных форматах, например, в виде ключ-значение, документов, графов.Используют специализированные языки запросов или API, которые отличаются от SQL.

Индексы:

Индекс-это объект БД, позволяющий быстрее выполнять запросы к БД. Индекс формируется из значений одного или нескольких столбцов и указателей на соответстующие строки.

Лямбда-функция

Лямбда-функция-это функция, которая не требует написания ключевого слова def, она же пишется в одну строку с ключевым словом lambda и используется для быстрого определения небольшой функции(также используются в качестве аргументов функции высшего порядка).

Пример:

double = lambda x: x*2

PEP - 8

Python Enhancement proposal 8

PEP - 8-это рекомендации по написанию кода, содержит стилевые соглашения, которые повышают читабельность кода, делая его более понятным, расширяемым и поддерживаемым.

SOLID

SOLID-это аббревиатура, обозначающая набор из пяти принципов объектно-ориентированного проектирования, которые помогают создавать более качественный и поддерживаемый код.


S - Single Responsibility Principle (Принцип единственной ответственности): Каждый класс должен иметь одну и только одну причину для изменения. Это значит, что класс должен выполнять одну конкретную задачу, и его код должен быть посвящен только этой задаче.
O - Open/Closed Principle (Принцип открытости/закрытости): Классы должны быть открыты для расширения, но закрыты для изменения. Это означает, что вы можете добавлять новую функциональность в класс, не изменяя его исходный код.
L - Liskov Substitution Principle (Принцип подстановки Лискова): Подтипы должны быть заменяемы на родительские типы без нарушения корректности программы. Это означает, что вы можете использовать подтип вместо родительского типа, не заботясь о том, что код будет работать неправильно.
I - Interface Segregation Principle (Принцип разделения интерфейса): Клиенты не должны зависеть от методов, которые им не нужны. Это означает, что вы должны разбивать интерфейсы на меньшие, более специфичные интерфейсы, чтобы клиенты использовали только те методы, которые им действительно нужны.
D - Dependency Inversion Principle (Принцип инверсии зависимостей): Модули верхнего уровня не должны зависеть от модулей нижнего уровня. Оба должны зависеть от абстракций. Абстракции не должны зависеть от деталей. Детали должны зависеть от абстракций.

def add_num(a, b):

Аннотации

Аннотации-это возможность указывать типы при объявлении переменных, полей класса, функций.
Для переменных-после имени переменной (str, int, list...)
Для параметров функций-синтаксис работает так же, но добавляется "->".

Пример:

Простые аннотации:

def greet(name: str) -> str:
"""Приветствует пользователя."""
return f"Привет, {name}!"

Аннотации для списков и словарей:

def sum_numbers(numbers: list[int]) -> int:
"""Суммирует числа в списке."""
total = 0
for number in numbers:
total += number
return total
